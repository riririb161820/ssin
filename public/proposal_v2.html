왜 이 플랫폼이 필요한가?

조선·기자재 업계의 고질적 문제

수기 발주 & 관리: 대부분의 거래가 여전히 전화/이메일 기반, 투명성 부족
외상대금 연체: 어음 기한 만기 시 현금 부족으로 자금 경색 심화
중소사 대출 애로: 담보 부족 및 신용평가 난제로 은행권 진입 불가
공급망 단절: 거래처 간 데이터 연계 미흡으로 리드타임 지연

공급망 금융(SCF) 플랫폼

디지털 발주: 클라우드 기반 주문·납품·검수의 완전 자동화
즉시 현금화: 발주 데이터 기반 '선정산' 상품으로 자금 순환 가속
Data 기반 신용평가: 거래 기록 누적으로 '씬파일러(Thin-filer)' 대출 가능
공급망 투명성: 전사 거래 흐름 가시화로 효율성 극대화

시장 진화 흐름

1세대: 단순 B2B 커머스 (2010~2015)

카페24, 고도몰 기반 상품 카탈로그 쇼핑몰 중심. IT 솔루션만 제공.

2세대: PG 연동 결제 솔루션 (2015~2020)

외부 PG사(토스, KCP) 에스크로와 B2B 신용카드 결제 도입. 여전히 은행 미연동.

3세대: SCF 내장형 플랫폼 (2020~현재)

금융기관이 플랫폼에 직접 결합하여 데이터 기반 씬파일러 대출 실행. 공급망 금융 최적화.

프로젝트 컨셉 & 요구사항

상상인 그룹의 IT·금융·제조 역량을 통합한다

프로젝트 타이틀

"공급망 금융(SCF) 내장형 B2B 기자재 마켓플레이스"
상상인그룹 계열사 시너지(IT + 금융 + 제조)를 극대화하여, 중소 조선·기자재 업체의 자금 경색을 해결하고 공급망 효율을 동시에 달성하는 통합 플랫폼

쿠팡 수준의 UX

B2B 발주 프로세스를 C2C 커머스 수준으로 단순화. 한 번의 클릭으로 대량 주문/결제 완료

실시간 여신 연동

상상인저축은행 API 연동으로 여신 한도 조회, 선정산(팩토링) 즉시 승인

수동 개입 백오피스

결제 장애(Edge Case) 발생 시 영업팀이 즉시 제어 가능한 Fallback 시스템

호스팅 제약 & 사내 구축 필수성

SaaS 방식 vs 자체 구축 및 Headless Commerce 전략

만약 호스팅사(SaaS)를 이용한다면?

✅ 장점 (Pros)

  • MVP 2~3개월 타임투마켓 단축
  • 장바구니/회원 관리 개발 공수 제로화
  • 초기 SI 비용 절감

❌ 한계 (Weakness)

  • 여신 DB 실시간 연동 차단
  • 복합 결제(Split Payment) UI 커스텀 불가
  • 고객 데이터 외부 서버에만 적재
  • API 트래픽 할당량 제한

필수 해결책: Headless Commerce 아키텍처

호스팅사를 이용하더라도 결제 단계의 Checkout 컴포넌트는 Headless 방식으로 구현하여, 금융 컴플라이언스와 망분리 규제를 완벽히 준수해야 합니다.
